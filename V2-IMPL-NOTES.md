# V2 Implementation Notes

## Key APIs to use:
- `ejb.getFunctionCodes()` → List<FunctionCodeInfo>
- `fci.getCode()` → "enrgCommande", "LSTCRTS"
- `fci.getEnumValue()` → "ENRG_COMMANDE"
- `fci.getDispatchPath()` → "flux/action" or "Flux/FONCTION"
- `fci.getInputFields()` → List<String> (paths like "flux/numAccount")
- `fci.getOutputFields()` → List<String> (paths like "flux/numCommande")
- `fci.isTestFunction()` → true if _TST suffix
- `fci.inferHttpMethod()` → GET/POST/PUT/DELETE
- `fci.deriveEndpointName()` → "enrg-commande", "lstcrts"
- `sourceMetadata.getJndiNameForEjb(ejbName)` → "ejb/DotationService" or null

## V2 Resource requirements:
- JNDI lookup via @PostConstruct + InitialContext.lookup("ejb/{jndiName}")
- One endpoint per FunctionCodeInfo (not per MethodInfo)
- @GET with @QueryParam when httpMethod==GET and inputFields.size() <= 3
- @POST when inputFields > 3 or httpMethod != GET
- _TST functions → separate test endpoints under /test/ sub-path
- No @EJB annotation, no "Generated by" signatures

## V2 Converter requirements:
- Build Envelope with setService(serviceName) + setMethod("process")
- Build Flux XML body: <Flux><FONCTION>code</FONCTION><field>value</field>...</Flux>
- Wrap in try/catch for ParsingException
- Map real response structure using fci.getOutputFields()
- Handle list/nested structures (paths like "rows/row,{i}/field")

## V2 DTO generation:
- For each FunctionCodeInfo, generate a Request DTO (from inputFields) and Response DTO (from outputFields)
- DTO prefix = capitalize(fci.deriveEndpointName().replace("-",""))
- Field names = extractFieldName(path) → last segment after "/"
- All fields are String type unless path analysis says otherwise

## Helper methods available:
- capitalize(str) → first char uppercase
- camelToKebab(str) → CamelCase to kebab-case
- extractFieldName(path) → last segment after "/"
- javaType(type) → maps to Java type
- isPrimitiveType(type) → checks if type is primitive

## EnvelopeField(path, fieldName, type) constructor
- path: full envelope path like "flux/numAccount"
- fieldName: Java field name like "numAccount"
- type: "String", "int", "boolean", "long", "double"

## Class structure:
- basePackage: the base Java package for generated code
- artifactId: Maven artifact ID
- groupId: Maven group ID
